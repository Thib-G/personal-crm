## Summary

Add a persistent sync status icon to the navigation bar that reflects four states (`synced`, `syncing`, `error`, `offline`) driven by the existing `SyncService` and browser connectivity events. This is a **frontend-only** change: no new backend endpoints, no new npm packages, no new IndexedDB tables. The icon is animated in the `syncing` state and exposes a tooltip in all states. State is held in a module-level Vue `ref` exported from `sync.ts` and read by a new `SyncStatusIcon.vue` component mounted in `App.vue`.

## Design Details

### sync.ts — changes

Export a reactive `ref<SyncStatus>` at module level:

```ts
export type SyncStatus = 'synced' | 'syncing' | 'error' | 'offline'
export const syncStatus = ref<SyncStatus>(
  navigator.onLine ? 'synced' : 'offline'
)
```

Update `SyncService`:

- `startSync()`: add `window.addEventListener('offline', () => { syncStatus.value = 'offline' })`
- `syncCycle()`:
  - Before any fetch: if `syncStatus.value !== 'offline'`, set `syncStatus.value = 'syncing'`
  - On successful completion (pull + push both ok): `syncStatus.value = 'synced'`
  - In the `catch` block: `syncStatus.value = navigator.onLine ? 'error' : 'offline'`
- `startSync()` already handles `'online'` event → `syncCycle()`, which will set `syncing` → `synced/error` naturally.

### SyncStatusIcon.vue — new component

Reads `syncStatus` from `sync.ts`. Renders a `<span>` with:
- A `title` attribute for the tooltip (browser-native, no JS needed)
- A CSS class per state for color
- A `@keyframes spin` CSS animation on the `syncing` state

| State | Icon character | Tooltip text | Color |
|---|---|---|---|
| `synced` | `✓` | "All data synced" | green |
| `syncing` | `↻` (spinning) | "Syncing…" | blue |
| `error` | `⚠` | "Sync failed — will retry" | red/amber |
| `offline` | `✗` | "Offline — sync paused" | grey |

### App.vue — changes

Import and place `<SyncStatusIcon />` in the nav bar, between the Settings link and the Logout button.
