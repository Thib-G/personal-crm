# Implementation Plan: Fix Logout — Route Guard and Session Invalidation

**Branch**: `007-fix-logout-guard` | **Date**: 2026-03-22 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/007-fix-logout-guard/spec.md`

## Summary

After logout, navigating to protected routes re-authenticates the user because the server session is not reliably invalidated (logout endpoint silently fails when Ninja's inherited auth check rejects the request) and the client-side Pinia state persists across in-app navigation. Fix: (1) remove the auth requirement from the logout endpoint, (2) perform a hard page reload after logout to purge client state, (3) redirect authenticated users away from the login page in the router guard.
